🔑 Authentication

All requests must include your API token using one of these methods (in priority order):

# Method 1: Bearer Token (recommended)
curl -H "Authorization: Bearer YOUR_TOKEN" \
     https://yourdomain.com/api/v1/stats

# Method 2: X-API-Token Header
curl -H "X-API-Token: YOUR_TOKEN" \
     https://yourdomain.com/api/v1/stats

# Method 3: Query Parameter (not recommended for production)
curl "https://yourdomain.com/api/v1/stats?api_token=YOUR_TOKEN"

🎫 How to Get Your API Token

Generate a token from your ERP dashboard:

  • Log in to your ERP dashboard
  • Go to Settings → API & Webhook
  • Click Create New Token
  • Set a name, expiry (optional), and abilities
Token abilities:
read — GET endpoints only
write — POST / PUT / PATCH
delete — DELETE endpoints
* — Full access (all abilities)

     https://yourdomain.com/api/v1/stats?period=2026-04

/api/v1/products
Paginated list of active products with stock per warehouse

     https://yourdomain.com/api/v1/products?page=1
POST
/api/v1/orders
Create a new sales order — requires write ability
curl -X POST \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{
  "customer_id": 12,
  "date": "2026-04-02",
  "notes": "Order via API",
  "items": [
    { "product_id": 5, "quantity": 10, "price": 15000 },
    { "product_id": 8, "quantity": 5,  "price": 25000 }
  ]
}' \
     https://yourdomain.com/api/v1/orders

📦 Response Format

All responses are wrapped in a standard envelope:

// Success
{
  "success": true,
  "message": "OK",
  "data": { /* payload */ }
}

// Error
{
  "success": false,
  "message": "Error description"
}

Paginated list endpoints include current_page, last_page, total, and next_page_url inside data.

🚦 Rate Limiting

READ GET 60 req/min Retry-After header
WRITE POST/PUT/PATCH 20 req/min Retry-After header

Limits scale with your subscription plan:

Plan Multiplier Read (eff.) Write (eff.)

Track usage via X-RateLimit-Limit and X-RateLimit-Remaining response headers.
